The Zen of Python

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
Special cases aren't special enough to break the rules.
Although practicality beats purity.
Errors should never pass silently.
Unless explicitly silenced.
In the face of ambiguity, refuse the temptation to guess.
There should be one– and preferably only one –obvious way to do it.
Although that way may not be obvious at first unless you're Dutch.
Now is better than never.
Although never is often better than right now.
If the implementation is hard to explain, it's a bad idea.
If the implementation is easy to explain, it may be a good idea.
Namespaces are one honking great idea – let's do more of those!

——— Tim Peters

Effective Python

Pythonic Thinking

  1. Use Python 3; Avoid Python 2.

  2. Follow PEP8 style guide.

  3. Understand differences between str and bytes.

  4. Learn str.format but prefer f-Strings (string interpolation).

  5. Wrap complex logic into helper functions.

  6. Prefer unpacking over indexing.

  7. Prefer enumerate over range.

  8. Use zip to process iterators in parallel.

  9. Avoid else blocks after for and while loops.

  10. Prevent repetition with assignment expressions (walrus operator :=).

List and Dictionaries

  1. [@11] Know how to slice sequences.

  2. Avoid striding and slicing in a single expression.

  3. Prefer catch-all unpacking (starred expression) over slicing

  4. Sort by complex criteria using key parameters

  5. Be cautious when relying on the dict insert order

  6. Prefer get over in and KeyError to handle missing dictionary keys.

  7. Prefer defaultdict over setdefault to handle missing items in internal states.

  8. Construct key-dependent default values with __missing__.

Functions

  1. [@19] Never unpack more than three variables when functions return multiple values.

  2. Prefer raising exceptions to returning None.

  3. Know how closures interact with variable scope.

  4. Reduce visual noise with variable positional arguments (varargs, starargs)

  5. Provide optional behavior with keyword arguments
